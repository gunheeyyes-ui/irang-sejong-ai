## 프로토타입에 포함된 것

### 화면 4개
| 탭 | 내용 |
| --- | --- |
| 홈 | 서비스 개요, 핵심 기능 3개 소개, 시범 범위 통계 |
| 외출 추천 | 자연어 + 폼 조건 입력 → 8개 기준 점수화 → 상위 3 + 대체안 |
| AI 질문 | 질문 카테고리 분류 → 공식 링크 + 응급 체크리스트 |
| 행정 대시보드 | 비식별 통계: 생활권/월령대/카테고리/공식링크 클릭률 |

### 데이터
- 모의 시설 **30개** (복합커뮤니티센터·도서관·공동육아나눔터·가족센터·박물관/체험관·공원·보건소)
- **세종시 직접 관리** vs **공식 페이지 연결** 구분 (제안서의 관리범위 구분 반영)
- 표준 정보 항목: 운영시간, 휴관일, 예약 필요 여부, 수유실, 기저귀 교환대, 세면·온수, 유모차 진입(일반/광폭), 엘리베이터, 주차, 비용, 혼잡도(평일/주말), 날씨 적합도, 월령 적합도, 정보 갱신일

### 점수 산정 (제안서 8개 기준 반영)
1. 영유아 편의 (수유실, 기저귀대, 세면·온수)
2. 이동 편의 (일반/광폭 유모차, 엘리베이터, 경사로)
3. 주차 편의 (실내 주차 가산)
4. 혼잡도 (평일/주말 자동 판별)
5. 비용 (무료 우대)
6. 운영시간 (오늘 휴관 자동 제외)
7. 환경 적합도 (우천/폭염/한파/미세먼지 매칭)
8. 월령 적합도 (시설별 권장 월령 매칭)

> "수유실 필요", "쌍둥이 유모차"처럼 **반드시 필요한 조건**은 hard-fail로 처리해 대체안 섹션으로 분리합니다.

### 자연어 이해 (간이 룰 기반)
- "8개월 쌍둥이와 비 오는 날 갈 만한 곳" 같은 문장에서
  월령·자녀 수·유모차 종류·날씨·시설 조건을 자동 추출합니다.
- 실제 운영 시에는 LLM API(예: Claude/GPT) 호출로 대체할 수 있는 분리된 구조입니다 (`parseFreeText` 함수).

### 비식별 로깅
- 검색·클릭·질문 이벤트를 브라우저 `localStorage`에 저장합니다 (개인정보 미수집).
- 행정 대시보드는 이 로그를 집계해 보여줍니다.
- **"시연용 샘플 데이터 추가"** 버튼으로 가상 60건+를 즉시 채워 발표용 차트를 보여줄 수 있습니다.

## 발표 시연 시나리오 (3분)

1. **홈 탭** — "이미 있는 공공정보를 오늘 조건에 맞게 연결하는 AI 안내 계층" 메시지 설명
2. **외출 추천 탭** — "8개월 쌍둥이 + 우천" 예시 버튼 클릭 → "AI 추천 받기"
   - AI가 이해한 조건 요약 → 추천 3곳 + 점수 근거 칩 → 대체안
   - 모든 추천에 **공식 안내 링크** + 정보 갱신일 표기
3. **AI 질문 탭** — "9개월 발열" 예시 클릭 → 응급 체크리스트 + 119/응급실 연결
   - "AI는 진단하지 않습니다" 안전장치 강조
4. **행정 대시보드 탭** — "시연용 샘플 데이터 추가" 클릭 → 생활권별 검색량, 월령대 분포, 수유실 검색 집중 등 차트 표시
   - "민원 발생 전 단계의 조기 수요 신호" 메시지 마무리

## 시범 사업과의 매핑

| 제안서 | 프로토타입 구현 |
| --- | --- |
| 1단계: 30개 시설 표준 정보표 | `js/data.js`에 30개 시설 정형화된 항목으로 구축 |
| 세종시 관리 vs 국공립 구분 | 카드 우측 `세종시 관리` / `공식 페이지 연결` 뱃지 |
| 8개 평가 기준 | `scoreFacility()` 함수에서 8개 항목 점수화 |
| 공식 페이지 병기 | 모든 카드 하단 "공식 안내 보기" 버튼 |
| 월 1회 갱신 / 정보 갱신 기준일 | 각 시설 `updatedAt` + 안내 문구 |
| 의료 판단 제외 | Q&A에서 응급/건강은 진단 없이 위험신호 + 공식 안내만 |
| 비식별 행정 데이터 | 생활권·월령대·카테고리 단위만 집계 (이름·주소 미수집) |
| 단독 정책결정 근거 아님 | 대시보드 하단 활용 안내 명시 |

## 초기 범위에서 제외된 기능 (의도적 제외)

제안서대로 다음 기능은 1단계 범위에서 제외했습니다.
- 실시간 예약 / 실시간 혼잡도
- 개인 간 돌봄 매칭
- 의료 진단 / 처방 제시
- 시민 제보 자동 반영 (담당자 검토 후 반영 구조 가정)

이 분리는 예산·운영 책임을 작게 시작하기 위한 의도이며, 2~4단계에서 점진적 도입을 검토합니다.
